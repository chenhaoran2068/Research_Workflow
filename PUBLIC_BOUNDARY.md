# Public Boundary

## Boundary Statement

This repository is a cleaned public workflow pack.

It is not a full public mirror of the local `Research` workflow runtime.

The public boundary exists to make the repository:

- truthful
- readable
- reusable
- safe to release without pretending that every local material must be public

## What Public `v1` Is Intended To Include

The public `v1` lane is intended to include:

- repository identity documents
- explicit scope and boundary documents
- a curated workflow reading spine
- a public-safe governance slice
- one explicit public-safe validation path
- a compact frozen evidence summary
- a bounded agent entry surface

## What Stays Local For Now

The following stay local unless later release decisions say otherwise:

- full raw experiment run folders
- full raw evidence packs
- workstation-specific validation and audit outputs
- full historical derivation archives
- local temporary worklists and drafting artifacts
- adjacent experiment programs not required for the first workflow-pack core

## Public-Boundary Interpretation Rule

If something is not present here, the safe interpretation is:

- it may be intentionally out of public scope
- it may still exist locally as source material
- it may be waiting for public-safe derivation rather than direct export

Do **not** assume:

- absence means nonexistence
- public omission means the local workflow lacks that surface
- this repository is trying to expose every local coordinate, every local
  evidence pack, or every local implementation surface

## Claim-Safety Rule

Safe public wording includes:

- "cleaned public workflow example"
- "public workflow-facing companion repository"
- "public workflow instantiation repository"
- "not a full runtime mirror"

Avoid wording such as:

- "full local system released"
- "complete public runtime"
- "entire evidence archive included"
- "nothing remains outside the repository"
