# Workflow Reading Order

## Purpose

This file gives the intended public reading order for this repository.

It is designed so that a new reader does not need to guess:

- what to read first
- what to read next
- which files are orientation files and which files are structural files

## First Pass: Orientation

Read these first:

1. `README.md`
2. `START_HERE.md`
3. `REPOSITORY_SCOPE.md`
4. `PUBLIC_STATUS.md`
5. `PUBLIC_BOUNDARY.md`
6. `RELATIONSHIP_TO_FRAMEWORK.md`

Goal of this pass:

- understand what this repository is
- understand what it is not
- understand the current public-release status

## Second Pass: Workflow Shape

Read these next:

1. `docs/workflow_overview.md`
2. `docs/workflow_map.md`

Goal of this pass:

- understand the workflow in plain route terms
- understand the current sealed-route shape
- understand the difference between the workflow lifecycle and the hardening
  route

## Third Pass: Governance Structure

Read these next:

1. `docs/public_registry_slice.md`
2. `docs/public_mechanism_map.md`

Goal of this pass:

- understand the minimum public governance skeleton
- understand how coordinates are organized
- understand how current mechanism identity is carried without claiming one
  separate executable per coordinate

## Fourth Pass: Validation, Agent Entry, And Evidence

Read these next:

1. `VALIDATION.md`
2. `AGENT_START.md`
3. `docs/frozen_results_summary.md`
4. `docs/evidence_claim_map.md`

Goal of this pass:

- validate the bundle
- begin bounded agent-assisted use
- inspect the compact frozen evidence layer

## Fifth Pass: Readiness Decision

Read:

1. `docs/v1_readiness_review.md`

Goal of this pass:

- understand why the repository is currently treated as `v1 ready`
- understand what remains outside first-release scope

## Reading Rule

If you are new here, do **not** start by trying to infer the whole workflow
from scattered technical details.

Instead:

- start from orientation
- then learn the workflow shape
- then learn the governance skeleton
- then move to validation and evidence
