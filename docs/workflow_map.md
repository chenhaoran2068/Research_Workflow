# Workflow Map

## Purpose

This file gives a public-safe map of the `Research` workflow.

It does two jobs:

1. show the workflow lifecycle in simple route terms
2. show the executed hardening route that led to the current sealed scope

## Lifecycle Map

```text
Problem intake
  -> Question formalization
  -> Knowledge readiness
  -> Route design / route lock
  -> Data resolution
  -> Build
  -> Envelope freeze
  -> Pilot
  -> Route disposition
      -> Continue
          -> Full analysis
          -> Claim calibration
          -> Writing / submission assembly
          -> Internal review
          -> Human review
          -> Final release QA
          -> External release / submission
      -> Bounded redesign
      -> Stop to archive
      -> Successor spawn
```

## Typed Return And Loop Map

The workflow keeps return movement explicit rather than informal.

Public-safe loop summary:

- `L1`: clarification return
- `L2`: knowledge return
- `L3`: data-resolution return
- `L4`: technical repair return
- `L5`: bounded redesign return
- `L6`: human revision return
- `L7`: successor-spawn return
- `L8`: post-submission reviewer return
- `L9`: stop-to-archive return

Interpretation rule:

- these are not casual retries
- they are typed lawful return paths

## Major Boundary Crossings

The workflow keeps stage-to-stage crossing explicit.

Public-safe crossing summary:

- intake to question
- question to readiness
- readiness to design
- design to route lock
- route lock to data resolution
- data resolution to build
- build to freeze
- freeze to pilot
- pilot to route disposition
- disposition to continue / stop / successor
- analysis to claim
- claim to writing
- writing to review
- review to release
- release to external output
- reviewer response to routed return
- terminal closure to durable outputs
