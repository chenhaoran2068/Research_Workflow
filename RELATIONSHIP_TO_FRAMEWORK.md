# Relationship To Framework

## Short Version

The two repositories do different jobs.

`Four-Layer-OCED-M-Framework`:

- explains the framework
- provides framework-level structure, terminology, schemas, templates, and
  minimal public-safe examples

`Research_Workflow`:

- is the workflow-facing companion repository
- is being authored as the main public workflow instantiation repository for
  the `Research` workflow
- is being authored to present a cleaned public workflow example

## Why There Are Two Repositories

We keep them separate so that:

- the framework repository can stay framework-first
- the workflow repository can stay workflow-first
- the public workflow pack does not get buried inside framework explanation
- the framework repository does not have to become a full workflow archive

## Current Wording Discipline

Safe current wording:

- `Research_Workflow` is the public workflow-facing companion repository to the
  framework repository
- `Research_Workflow` is being prepared as the main public workflow
  instantiation repository for `Research`
- `Research_Workflow` is being authored to present a cleaned public workflow
  example

Avoid stronger wording for now:

- do not yet call this repository a full public workflow runtime mirror
- do not yet call it a complete public workflow example package unless `v1`
  readiness has actually been reached
- do not collapse the framework repository and workflow repository into one
  claim
