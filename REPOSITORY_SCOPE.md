# Repository Scope

## Scope Statement

`Research_Workflow` is being authored as a public workflow pack for the
`Research` workflow.

Its job is to expose a cleaned, public-safe workflow-facing surface that can
later be:

- read directly by humans
- validated through explicit public-safe commands
- used in bounded form with agent hosts such as `Codex CLI` or `Claude Code`

## Intended `v1` Scope

`v1` is intentionally narrower than a full runtime release.

The target scope is:

- repository identity and reading entry
- explicit public boundary statements
- a curated workflow reading spine
- a minimum public-safe governance slice
- one explicit validation path
- a compact frozen evidence summary
- a bounded agent entry surface

## Out Of Scope For `v1`

The following are outside first-release scope:

- full local runtime mirroring
- workstation-specific path bindings
- raw experiment run archives
- full historical derivation archives
- paper workspace artifacts
- every local governance or evidence object in unfiltered form

## Scope Discipline

This repository should be described as:

- a cleaned public workflow pack
- a workflow-facing companion repository to the framework repository
- a public-safe workflow instantiation lane

This repository should **not** be described as:

- the full local research system
- a complete public runtime mirror
- the paper's complete evidence archive
- proof that no future reratification or bug-fix work will ever be needed
